# 📊 Bithumb Auto-Scalper 전략 보고서 (v1.0)
**ChoiGPT Corp. 수석 소프트웨어 아키텍트 및 전략가 팀**

## 1. 개요 (Executive Summary)
본 보고서는 빗썸 API 2.0(JWT) 기반으로 구축된 **초단타 스캘핑 엔진(Auto-Scalper)**의 핵심 알고리즘과 리스크 관리 체계를 정의합니다. 본 엔진은 사용자님의 1,000,000 KRW 자산을 운용하며, 세력의 매집 신호를 포착하여 짧은 주기의 복리 수익을 창출하는 것을 목표로 합니다.

---

## 2. 모니터링 및 종목 선정 알고리즘 (Filtering Logic)
세력의 장난(자전거래)을 회피하고 실제 유동성이 확보된 종목만을 타겟팅합니다.

*   **유동성 필터 (Liquidity Threshold)**:
    *   24시간 거래대금 **300억 원 이상** 종목으로 제한.
    *   이유: 슬리피지(Slippage)를 최소화하고 즉각적인 엑싯(Exit) 유동성을 보장하기 위함.
*   **매집 포착 시그널 (Accumulation Detection)**:
    *   가속도 분석: 거래량은 전일 대비 200% 이상 폭증했으나, 가격 변동폭이 **-1% ~ +1%** 내외인 종목 추출.
    *   이유: 세력들이 대량 매집 시 가격을 급격히 올리지 않고 물량을 확보하는 구간(Sideways Accumulation)을 공략.
*   **안티-매니퓰레이션 (Anti-Manipulation)**:
    *   **자전거래 필터**: `체결 건수 / 총 거래량` 비율을 분석하여 인위적인 거래량 부풀리기 포착 시 진입 차단.
    *   **바이낸스 동기화**: 글로벌 시세(Binance SUI/USDT 등)와 빗썸 시세의 괴리가 5% 이상일 경우(비정상 펌핑) 진입 보류.

---

## 3. 매수/매도 실행 알고리즘 (Execution Engine)

### A. 매수 로직 (Entry Strategy)
*   **분산 투자**: 총 예산 100만 원을 **20만 원씩 5슬롯**으로 분할 운용.
*   **진입 방식**: 지정가(Limit Order) 매수. 호가창 상단 3호가 내의 두꺼운 매수벽 위에 전략적으로 배치.
*   **인증 보안**: Bithumb API 2.0 규격에 따른 **JWT(HS256) 토큰 인증** 및 쿼리 해싱(Query Hashing) 적용으로 전송 구간 보안 극대화.

### B. 매도 로직 (Exit Strategy)
*   **고정 익절 (Take Profit)**: 진입가 대비 **+1.5%** 도달 시 즉시 시장가 혹은 우선순위 지정가 매도.
*   **강제 손절 (Stop Loss)**: 진입가 대비 **-1.0%** 하락 시 망설임 없이 손절 처리하여 자산 방어.
*   **시간 제한 (Time Cut)**: 진입 후 4시간 이내에 목표 수익률에 도달하지 못할 경우 기회비용 확보를 위해 전량 청산.

## 4. 인프라 및 가동 환경
*   **플랫폼**: GitHub Actions (24/7 Cloud Execution).
*   **주기**: 5분 간격 스케줄링 가동 (무료 환경 최적화).
*   **모니터링**: GitHub Pages 실시간 대시보드를 통해 보유 포지션 및 AI 감시 종목을 실시간 송출.

## 5. 향후 고도화 계획 (Roadmap)
1.  **Trailing Stop 도입**: 수익 발생 시 손절선을 따라 올리며 수익 극대화.
2.  **소셜 데이터 결합**: X(구 트위터)의 특정 키워드 언급량과 거래량 상관관계 분석 모듈 통합.
3.  **바이낸스 선물 헤징**: 빗썸 매수 시 바이낸스에서 동일 액수의 숏(Short) 포지션을 잡아 시장 급락 리스크 중화.
